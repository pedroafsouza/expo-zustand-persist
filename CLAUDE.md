# expo-zustand-persist

A drop-in `persist` middleware for Zustand that works with React Native / Expo (AsyncStorage, SecureStore, etc.).

## Project structure

- `src/Persist.ts` — The entire library: `persist`, `createJSONStorage`, and all supporting types.
- `src/index.ts` — Re-exports everything from `Persist.ts`.
- `dist/` — Built output (CommonJS + declarations). Generated by `npm run build`.

## Key types

- **`PersistImpl`** — Generic over zustand's `StoreMutatorIdentifier` arrays (`Mps`, `Mcs`) so `persist` composes with any middleware stack (devtools, immer, subscribeWithSelector, etc.).
- **`PersistOptions<S, PersistedState>`** — Config: storage backend, partialize, migrate, merge, version, skipHydration.
- **`PersistStorage<S>`** — Adapter interface (`getItem`/`setItem`/`removeItem`) that storage backends must implement.
- **`StateStorage`** — Raw string-based storage interface (what `createJSONStorage` wraps).

## Commands

- `npm run build` — Compile TypeScript to `dist/` (runs `tsc`).
- `npx tsc --noEmit` — Type-check without emitting.
- No test suite exists yet (`npm test` is a placeholder).

## Design notes

- The implementation function (`persistImpl`) is concretely typed with `Record<string, unknown>` internally and cast to the generic `PersistImpl` at the export boundary. This mirrors zustand's own pattern — the middleware type parameters can't be inferred inside the implementation body.
- `toThenable` wraps sync/async storage calls into a uniform `.then/.catch` chain so the hydration logic works with both sync and async storage backends.
- `zustand` is a regular dependency (^5.0.0). The package targets ES6 / CommonJS.
